# Skill Review Checklist

## Metadata

- [ ] `name` is clear and follows naming conventions
- [ ] `description` covers WHAT + WHEN
- [ ] `version`, `owner`, `risk` are filled in
- [ ] `triggers_intent` and `triggers_api` are both declared
- [ ] `scope`, `out_of_scope`, `depends_on` are declared

## Content

- [ ] Contains all 7 required sections
- [ ] `When to Use` and `When NOT to Use` are unambiguous
- [ ] `Workflow` steps are actionable
- [ ] `API Matrix` matches the declared scope
- [ ] Includes Minimal Request/Response Pattern
- [ ] `Error Handling` is actionable
- [ ] `Security Notes` matches the risk level

## Boundaries

- [ ] Does not significantly overlap with other skills' `scope`
- [ ] `out_of_scope` is sufficiently explicit
- [ ] `depends_on` only expresses suggestions, not runtime hard dependencies

## Examples

- [ ] `examples.md` exists
- [ ] At least 1 end-to-end example
- [ ] Examples are consistent with section descriptions
